## Dev Notes

### Arquitetura de Referência

O TravelMatch BR é uma plataforma fullstack serverless-first:
- **Frontend:** Next.js 16+ (App Router) com TypeScript strict, Tailwind CSS 4, shadcn/ui
- **Backend:** Supabase (PostgreSQL 15+ com pgvector) + Edge Functions (Deno)
- **Monorepo:** pnpm workspaces — `packages/web` (PWA), `packages/shared` (types/contracts)
- **Deploy:** Vercel (web) + Supabase (DB, Auth, Functions)

**Nota:** `packages/admin` e `packages/ai-engine` serão criados em stories futuras (Epic 4+). Nesta story, criar APENAS `packages/web` e `packages/shared`.

### Migrations Existentes

O schema completo do banco já está criado em `supabase/migrations/` (10 arquivos, 16 tabelas). Esta story NÃO precisa criar migrations — apenas aplicá-las via `supabase db push`.

Tabelas criadas:
- `profiles` — Perfis de usuário (extends auth.users)
- `dna_profiles` — DNA de viagem com VECTOR(10) e HNSW index
- `quiz_responses` — Respostas do quiz com resumabilidade
- `dna_history` — Histórico de evolução do DNA
- `destinations` — Destinos com VECTOR(10) e HNSW index
- `destination_scores` — Scores por dimensão (10 dims)
- `destination_weather` — Cache de clima mensal
- `destination_requirements` — Documentos, taxas, vacinas
- `partners` — Parceiros curados com Asaas wallet
- `packages` — Pacotes de viagem personalizados
- `package_items` — Itens do pacote
- `package_checklist_items` — Checklist pré-viagem
- `package_payments` — Pagamentos Asaas (Pix, cartão, boleto)
- `payment_split_items` — Split por parceiro
- `recommendation_cache` — Cache de recomendações
- `reviews` — Avaliações com NPS

### Seed Data

Disponível em `supabase/seed/`:
- `destinations.sql` — 10 destinos brasileiros reais com vetores e scores
- `admin-user.sql` — Promoção de usuário admin

### Supabase Client Pattern

Usar o padrão Supabase SSR (não o antigo `createClient`):
- **Browser:** `createBrowserClient()` — para Client Components
- **Server:** `createServerClient()` com cookies — para Server Components / Server Actions
- **Middleware:** `createServerClient()` com request/response — para Edge Middleware

### Variáveis de Ambiente

Referência completa em `docs/architecture/fullstack-architecture.md` Section 13.4.

Variáveis necessárias nesta story:
- `NEXT_PUBLIC_SUPABASE_URL` — URL do Supabase local (`http://localhost:54321`)
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` — Chave anon do Supabase local
- `NEXT_PUBLIC_SENTRY_DSN` — DSN do Sentry (pode ser vazio em dev)
