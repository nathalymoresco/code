## Acceptance Criteria

1. Package `packages/admin` com Next.js 15+, Tailwind CSS 4, shadcn/ui base components
2. Auth admin via Supabase com verificação de role (`is_admin()` já existe no DB)
3. RLS: admin acessa todas tabelas de catálogo (policies já existem nas migrations)
4. Sidebar responsiva: Dashboard, Destinos, Parceiros, Configurações
5. Dashboard com métricas placeholder (total users, total DNAs, card vazio de destinos)
6. Deploy target: `admin.travelmatch.com.br` (Vercel, separado do web)
7. Middleware de proteção: redireciona não-admin para página de acesso negado
8. Seed script: 1 usuário admin para desenvolvimento

## Dev Notes

### Existing Infrastructure

- **Supabase migrations**: 00001-00010 já existem com schema completo
- **`is_admin()` function**: já existe em `00009_rls_policies.sql` — verifica `profiles.role = 'admin'`
- **RLS policies admin**: já existem para profiles, destinations, partners, packages, payments, reviews
- **profiles.role**: coluna já existe com CHECK (`'user'`, `'admin'`), default `'user'`
- **pnpm-workspace.yaml**: `packages/*` — admin será detectado automaticamente
- **tsconfig.base.json**: config base compartilhada entre packages

### Architecture Decisions

- Admin é package **separado** do web (não subpasta) — deploy independente na Vercel
- Dev port: 3001 (web usa 3000)
- Mesmo Supabase instance, mesmas env vars
- Auth: email/password (admin não usa OAuth/magic link por segurança)
- Middleware verifica role no DB, não em JWT claims (mais seguro, menos performático mas OK para admin)

### Design References

- Cores: mesmos design tokens do web (sand, coral, turquoise)
- Sidebar: padrão admin dark (sand-800/900) com hover turquoise
- Ícones: lucide-react (mesmo do web)
- Layout: sidebar 256px fixa + content fluid

### Web Package Reference

- `packages/web/src/lib/supabase/server.ts` — padrão createClient
- `packages/web/src/lib/supabase/middleware.ts` — padrão updateSession
- `packages/web/src/middleware.ts` — padrão route protection
- `packages/web/vitest.config.ts` — padrão vitest setup

### Testing Strategy

- Unit: Vitest + RTL (sidebar, dashboard, login)
- Test location: `packages/admin/src/**/__tests__/*.test.ts`
- Mocks: Supabase client mocado para testes de auth/middleware
