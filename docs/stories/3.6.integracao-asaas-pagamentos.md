## Story

**As a** usuário que finalizou pacote,
**I want** pagar com Pix, cartão ou boleto de forma segura,
**so that** eu confirme minha viagem com confiança.

---

## Acceptance Criteria

1. [x] Asaas client tipado com createCustomer, createPayment, getPixQrCode
2. [x] API route `POST /api/payments/create`: cria customer + cobrança Asaas
3. [x] Pix (QR code + copia-e-cola, 30 min), Cartão (até 12x), Boleto (3 dias úteis)
4. [x] Escrow: pagamento retido, liberado ao parceiro pós-viagem (end_date + 3 dias)
5. [x] Webhook handler: PAYMENT_CONFIRMED, OVERDUE, REFUNDED → atualiza status
6. [x] Checkout page com seleção de método, dados do cliente, submit
7. [x] Página de confirmação pós-pagamento com status dinâmico
8. [x] Validação server-side, anti-duplicidade (verifica pagamento existente)
9. [x] Status mapping Asaas → interno (9 status mapeados)
10. [x] Testes: 34 cenários (21 checkout UI + 13 API utils)
