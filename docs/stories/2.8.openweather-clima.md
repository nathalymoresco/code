## Technical Notes

- OpenWeather API key via `OPENWEATHER_API_KEY` env var (opcional)
- Sem API key: usa fallback estático baseado em latitude (tropical vs subtropical)
- Cache strategy: DB-first (check destination_weather), API call apenas se stale
- Rate limiter in-memory (Map) — suficiente para single-instance MVP
- Widget é non-critical: falha silenciosamente sem impactar a página
- Modelo estático: Oct-Mar chuvoso, Jun-Aug mais frio, baseTemp por latitude
