## Story

**As a** sistema,
**I want** calcular score de compatibilidade DNA ↔ destino,
**so that** recomendações sejam ordenadas por relevância.

---

## Acceptance Criteria

1. [x] API Route `GET /api/compatibility` — retorna destinos com score 0-100%
2. [x] Chama RPC `match_destinations` (cosine similarity via pgvector)
3. [x] Ajuste de sazonalidade: +5% se mês atual está em best_months
4. [x] Ajuste de budget: penaliza se avg_daily_cost fora do range do perfil
5. [x] Cache em `recommendation_cache` com TTL 24h, hash do DNA vector
6. [x] Fallback: retorna destinos por popularidade se DNA indisponível
7. [x] Performance: < 500ms para 20 destinos
8. [x] Testes: 5+ cenários de matching
9. [x] Endpoint acessível no frontend via hook `useCompatibility`
10. [x] Types exportados em shared
